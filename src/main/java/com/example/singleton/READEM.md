单例模式常见实现方式的区别总结：

1. 三种饿汉式的区别

- 静态常量：类加载时直接初始化实例，代码最简单，类加载即可用，但无法做到按需延迟初始化。
- 静态代码块：类加载时在静态代码块中初始化实例，支持更复杂的初始化逻辑（如异常处理或多步创建），仍是类加载即创建，无法延迟。
- 枚举：类加载时直接初始化实例，代码最简单，类加载即可用，但无法做到按需延迟初始化。

2. 四种懒汉式的区别

- 普通懒汉式（非线程安全）：首次调用时创建实例，单线程性能好，但多线程下可能创建多个实例。
- 同步方法懒汉式：通过方法级同步保证线程安全，实现简单但每次获取都加锁，性能一般。
- 双重检查锁（DCL）：先检查再同步，减少锁竞争，需要配合 `volatile` 防止指令重排，性能与安全性平衡。
- 静态内部类：利用类加载机制按需初始化，线程安全且无显式同步，代码简洁、性能好。

3. 饿汉式和懒汉式的区别

- 初始化时机：饿汉式在类加载时创建，懒汉式在首次使用时创建。
- 资源占用：饿汉式可能提前占用资源，懒汉式按需分配更省资源。
- 线程安全：饿汉式天然线程安全，懒汉式需额外措施确保线程安全。
